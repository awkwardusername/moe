Chapter 1: Introduction

 

 

MOE is an educational experimental programming language. It has been inspired by the tactful minimalism of the C programming language, assembly (as for the short, albeit abbreviated reserved words), Perl, and a bit of creativity. Primitive in its nature, the MOE programming language attains to be a complete programming language to be used in practical and elementary programming. As a typed language, it also follows strict but natural syntax with recommendation on the implementation of structure and readability.

MOE provides scope, control structures, function and iteration. Having a built in input/output, a natural process for looping and decision making, an unambiguous assignment operator (e:f instead of e=f), every statement is straightforward.

Being a C-like programming language, identifiers are case-sensitive and blocks are indicated by curly brackets. The program logic starts in an up-down manner, and programs are executed first in the run() function (like Cfs main()), therefore, a run() function is always required.

MOE aims to be a kick-starter for elementary level programmers where basic knowledge of programming is required. As MOE is a minimal language, it is easier to learn by beginners because of the limited and strict syntax of the code, but powerful enough to implement algorithms into.

Technically, a MOE program is interpreted by a fellow C compiler in which the system is mostly based on. MOE has no external libraries as all operations are done in an interpreter.

1.1     Getting Started

As it was quoted in gThe C Programming Languageh, the only way to learn a new programming language is by writing programs in it. The first program to write is the same for all languages

Print the words

hello, world

In MOE, the program to print ehello, worldf is:

      run() {

            prnt ehello, worldf;

}

 

NOTE: String literals in MOE are enclosed in single quotes. Otherwise, they arenft.

1.2    Variables and Arithmetic Functions

The next program will demonstrate the basic functionalities of the MOE programming language by printing a table of equivalent Fahrenheit and centigrade/Celsius values:

   1    -17

   20   -6

   40   4

   60   15

   80   26

   100  37

   120  48

   140  60

   160  71

   180  82

   200  93

   220  104

   240  115

   260  126

   280  137

   300  148

 

The program in this example is longer than the one that can print ghello, worldh, but it is not complicated. It introduces several new ideas including comments, declarations, variables, arithmetic expressions, loops, and output.

// print Fahrenheit-Celsius table

// for fahr = 0, 20, c, 300

 

run() {

      num fahr, celsius;

      num lower, upper, step;

 

      lower : 0;

      upper : 300;

      step : 20;

 

      fahr = lower;

 

      rpt is fahr =< upper ? {

            celsius : 5 * (fahr-32) / 9;

            prnt fahr + e f + celsius;

            fahr : fahr + step;

}

}

 

 

 

The two lines

// print Fahrenheit-Celsius table

// for fahr = 0, 20, c, 300

 

are a comment, which in this case explains briefly what the program does. Any characters after the symbols, //, are ignored by the compiler; they are often used to make the source code easier to read. A comment can appear anywhere a blank, a tab, or a newline exists.

In MOE, all variables are declared before they are used, usually at the beginning of a function before any executable statements. A declaration announces the properties of variables; it consists of a name and a list of variables such as

      num fahr, celsius;

num lower, upper, step;

 

NOTE: The rules of variable declaration are consistent with C. That is, an identifier must not start with a number or a symbol (operators, control characters, special symbols), should not be a reserved word (refer to section 1.6), and is composed of members of the ASCII character set. That is, the following identifiers are valid:

      ident, number, _something, print, CONSTANT

and the following are not valid:

      num, str, STR, CNST, is, 9variable, /string

 

Please take into note that you still cannot use the uppercase version of a reserved word since MOE is case sensitive. There is also a 16 character limit as to naming an identifier.

Also, we have to put into mind that declarations are needed to be placed before the statements. This complies with the C standard of placing variable declarations before everything in the program to assure that they are declared before being used.

From where we left off, the type num means that the variables listed are integers, in which by contrast with flt, which means floating point, i.e., numbers that has a fractional part.

Computation in the temperature conversion program begins with the assignment statements

      lower : 0;

      upper : 300;

      step  : 20;

 

which set the variables to their initial values. It is important that variables are to be initialized (except in a string variable) before they are used. Individual statements are terminated by semicolons.

1.3    Looping and conditional structures

If we see the previous code example, each line of the table is computed the same way, so we use a loop that repeats once per output line; this is the purpose of the rpt is loop. This loop is similar to the while loop used in many C-like programming languages. Although simple in nature, the rpt is loop is actually a form of the is conditional construct. Let us see this example which prints the string eyesf if 3 is greater than 5:

      run() {

            str answer : eyesf;

            is 3 < 5 ?

                  prnt answer;

      }

 

while the looping construct is formed as shown in this program that will print eyesf while the variable incr is still less than 5:

      run() {

            num incr : 0;

str answer : eyesf;

 

            rpt is incr < 5 ? {

                  prnt answer;

                  incr : incr + 1;

}

      }

 

We can see the inherent similarities in structure and format of an if conditional structure and a while loop as it was in C-like programming languages. A C for loop is made by utilizing an external iterable value (a num). It is important to note that conditional structures are not enclosed in parentheses and is always ended by a question mark (?). As with C control structures, a rpt is loop and is conditional can be nested and followed by blocks (curly braces) for scope control as shown in this short (albeit awkward) program that checks if 5 is less than 10 and if that is the case, checks if the variable ten is equal to 10, and if that is true, prints ten asterisks (*) on a row.

      run() {

            num itr : 0;

            num ten : 10;

           

            is 5 < 10 ? {

                  is ten = 10 ? {

                        rpt is itr < 10 ? {

                              prnt e*f;

                              itr : itr + 1;

                        }

                  }

            }

}

 

We can also tell the program to do something else if the conditionals are not satisfied (as the else case in C). Let us modify the program to say something if it has finished printing the asterisks.

      run() {

            num itr : 0;

            num ten : 10;

           

            is 5 < 10 ? {

                  is ten = 10 ? {

                        rpt is itr < 10 ? {

                              prnt e*f;

                              itr : itr + 1;

                        } no is itr = 10 ? {

                              prnt e\nDONE!f;

                        }

                  }

            }

}

 

We can see in this example that we could add a construct functioning like eelsef from C as an otherwise-function if the condition was not met. This is a unique feature of MOE programming language as it gives the power of an iterator-conditional an alternate code logic if the condition was not satisfied. This makes it more natural to read as the actual application for real world problems can be simulated by the MOE programming language (as it was in this case).

As it was in C, the compiler doesnft care how the code would look like, but the MOE programming language recommends using strict code constructs, indentation, and hierarchy.

1.4    Constants

It is bad practice to place magic numbers in a program (i.e. variables declared with numbers with no specific meaning) like what we have done with the temperature conversion table with making 300 as the upper limit, 0 as the lower limit, and 20 as the step size of each entry in the table.

MOE helps the user alleviate this problem by providing the keyword cnst for constant value declaration. In this case:

run() {

      num fahr, celsius;

      cnst num LOWER : 0;

cnst num UPPER : 300;

cnst num STEP : 20;

 

      // lower    : 0;

      // upper    : 300;

      // step     : 20;

 

      fahr = LOWER;

 

      rpt is fahr =< UPPER ? {

            celsius : 5 * (fahr-32) / 9;

            prnt fahr + e f + celsius;

            fahr : fahr + STEP;

}

}

 

We can see that there are numerous differences from our previous code as in the declaration of the variables (LOWER, UPPER, and STEP). cnst requires the user to set as all caps the identifiers and initialize the constants; otherwise, a compiler error will be produced.

1.5    Input and output

If it was tricky in C, pretty easy in Java and C#, and laidback in Perl, PHP, strings, input and output are effortless in the MOE programming language. As an elementary, minimalistic programming language, there is a need to have a native support for these equally important elements.

      run() {

            str answer;      

            inpt answer;     

prnt answer;     

      }

Now, let us discuss what the code does, for dummies.

str answer;

 

This statement declares a string variable answer which doesnft specifically need any initialized values (Note: it is for the reason that declaring a string variable automatically initializes the variable. This only applies to lists elstf and strings estrf). Moving on

inpt answer;

 

It may be confusing at first time, but just think of it this way. inpt takes the same logic as scanf in C, with the console waiting for user input, and the pressing of the carriage return marks the end of the input stream and will try to store that stream into the variable. The very big difference is, the inpt statement will automatically determine the data type of the variable and will try to check for invalid inputs. We do not need to explain this for now, but keep in mind that in this case, if the variable declared was an integer or a float value, keying in a character would make the program alert you for a data type mismatch. And of course

      prnt answer;

 

which will print the contents of the variable answer to the console.

A larger example of the I/O statements is shown below

      // this program will demonstrate the division of two real numerals

 

      run() {

            flt   divisor, dividend, quotient;  // declaration of the

                                                //    operands

            prnt eInput the dividend: f;

                  inpt dividend;                // indented to clarify

//    prompts

            prnt eInput the divisor: f;

                  inpt divisor;

 

            quotient : dividend / divisor;      // divides the dividend to

                                                // the divisor and stores it

// in the variable quotient

 

            prnt quotient;                      // print to console

      }

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Chapter 2: Formalities

 

 

2.1 Program structure

A typical MOE program is saved in a text file. The program structure is usually

      run() {

            <statement1>;

            <statement2>;

            c

}

 

where the run() keyword denotes the start of a program and also for denoting a MOE source code, the braces which define scope (an opening brace for the start and the closing brace for program termination) where a statement can be any kind of statement (declaration, assignment, conditional, control, function call).

As stated, the compiler will determine MOE source code by the program structure. It will ignore any file type the text file was saved in.

2.2 Data types

There are six fundamental data types in MOE. Four basic data types; num, flt, str, bool, and the abstract data types (experimental) lst, and arry.

2.2.1 num

A num datatype can hold values of long int in C which typically range from -2147483648 to 2147483648. Trying to store any other value other than mentioned will generate an error (either at runtime or compilation).

2.2.2 flt

A flt datatype can hold values of double in C which typically and approximately range from 1.0 x 10-38 to 1.0 x 1038. Storing an integer in it is allowed but storing characters and strings will generate an error (either at runtime or compilation).

2.2.3 str

A str datatype can hold characters and strings just like in C#, Perl, and others. To store a string in a str variable in coding, it must be enclosed in single quotes, otherwise, it will generate a compiler error. In runtime, a str variable will accept any input in it as it will convert the keystream to a string literal, regardless of the intended type it was for use. That is, it is safe to key in a number in a str variable but it will be regarded of as a string.

2.2.4 bool

A bool data type can hold either the values true or flse, otherwise, it will generate an error. In MOE, the result of a conditional statement is a bool.

NOTE: the bool data type cannot be used as a variable for input.

2.2.5 lst (experimental)

A lst data type is a list abstract data type. It is actually a doubly linked list. A lst holds a specific data type of which it is unbounded by the limits of a typical array. A lst could be treated as a stack or a queue at the same time. Operations on a list are the following:

·         add – adds the value to the end of the list

·         push – adds the value to the top/first element of the list

·         pop – gets the value at the top/first element of the list

·         eqe – adds the value at the end of the list (same with add)

·         dqe – gets the value at the top/first element of the list (same with pop)

 

2.2.6 arry (experimental)

An arry data type is an array abstract data type. Arrays are fixed in size and is iterable. The lst data type is preferred in one dimensional lists though.
